# Vibe Tree v0.7.0 发布文案草案

目标版本：`v0.7.0`。

这份文档分成两部分：

- App 内更新公告：来自 `updates/manifest.json`，给普通用户快速理解本次更新。
- GitHub Release Notes：正常 release 记录，给想了解完整版本变化的用户和开发者看。

## App 内更新公告

App 内公告不直接展示完整 GitHub Release body，而是读取 `updates/manifest.json` 里对应版本的结构化内容。

当前 `v0.7.0` 的 App 内展示内容大致如下，发布前需要用户确认：

```text
Vibe Tree v0.7.0

本次更新加入好友与群组社交页、macOS 菜单栏小窗，并继续打磨多设备同养和排行榜体验。升级后，原来的本机小树会继续保留；不登录 GitHub 也可以继续只在本机使用。

主要变化
- 新增社交页，可以添加好友、创建或加入群组，一起查看成长和排行。
- macOS 新增菜单栏小树入口，可以从状态栏快速查看成长、来源、排行榜和同步状态。
- 菜单栏浮窗支持在设置中选择显示哪些组件，并调整顺序。
- 登录同一个 GitHub 账号后，可以在多台设备共同养一棵树；其他设备的数据会按真实来源合并展示。
- 排行榜提供 24h、7 天、30 天和全部视角，并支持可选每小时自动同步。

体验改进
- 设置页继续整理，同步、更新、路径和浮窗组件等选项更容易找到。
- 加入排行榜确认、同步入口、设备贡献和升级提示等交互做了打磨。

隐私说明
- 社交、排行榜和同养小树都是可选功能。
- Vibe Tree 不会上传提示词、回复内容、代码文件、本地路径或会话正文。

升级说明
- 升级不会清空本机数据。
- 如果 macOS 菜单栏图标暂时看不到，可能是系统菜单栏空间不足，可以先收起不常用的状态栏图标。
```

## GitHub Release Notes

下面是建议放进 GitHub Release body 的正式版本记录。

```md
# Vibe Tree v0.7.0

## Added

- Added a Social page with friends, groups, group invites, group leaderboards, and relationship-aware profile cards.
- Added a macOS menu bar tree with a native status item and a focused popover for live growth, sources, leaderboards, sync status, and recent activity.
- Added menu bar component controls in Settings so users can choose and reorder what appears in the popover.
- Added clearer device contribution summaries for shared trees.
- Added optional hourly auto-sync controls for shared-tree sync and leaderboard publishing.

## Changed

- Reorganized Settings so sync, updates, source paths, and menu bar component choices are easier to find.
- Clarified the difference between private shared-tree sync and public leaderboard publishing.
- Refined leaderboard confirmation, refresh, join, leave, and usage-preference sharing flows.
- Synced data is merged back into real sources such as Codex, OpenClaw, and Claude Code instead of appearing as a separate cloud source.
- Refined the `24h / 7d / 30d / all` leaderboard ranges, including the rolling 24h view.
- Level-up notices, sync status, and device contribution displays have more stable placement and clearer wording.

## Privacy

- Social features, shared-tree sync, leaderboard publishing, and public usage preferences are optional.
- Shared tree sync sends only growth data needed for the tree, such as token counts, device id, coarse platform, achievement state, source category, and daily aggregate model totals.
- Leaderboard sync sends daily token totals, rolling 24h hourly aggregates, and local first-use date.
- Public usage preferences, when enabled, send only aggregate agent, model, time-period, and peak-token summaries.
- Vibe Tree does not upload prompts, replies, code file contents, local paths, session text, hostnames, or local usernames.

## Compatibility

- Existing local trees continue to work after upgrading.
- Users who do not enable shared-tree sync can keep using Vibe Tree locally as before.
- Existing leaderboard `today` requests remain supported as a compatibility alias for `24h`.
- macOS menu bar icons can be hidden by the system when the menu bar is crowded; the app keeps the Dock and manager window available.
```

## 发布时需要确认

- `package.json` 版本号和 tag 是否已经更新到目标版本。
- `updates/manifest.json` 里的 `version` 和 `fullReleaseUrl` 是否对应真实 release tag。
- GitHub Release body 是否使用上面的正常 Release Notes，而不是 App 内公告。
- App 内「查看当前版本内容」是否能显示 manifest 中的短公告。
- 发布前需要用户确认 App 内短公告。
