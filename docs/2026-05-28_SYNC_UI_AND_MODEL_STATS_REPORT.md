# 2026-05-28 同养小树同步收尾记录

## 背景

这次收尾的目标不是继续扩大同步功能，而是把已经接入的多设备同步做成用户能理解的产品语义：

- 首页统计只展示真实来源，例如 Codex、OpenClaw、Claude Code。
- 云端拉回来的事件应该合并回原来源，不再显示成单独的云端小树来源。
- 设置里的同步、刷新、加入、退出按钮要和实际后台动作一致。
- 没有开启同养小树的单设备用户，不应该被新增的云同步设置打扰。
- 隐私边界不变：不上传提示词、回复、代码文件、本地路径或会话正文。

## 今晚完成的修改

### 首页统计来源

- 隐藏了统计页里的 Cloud Tree 独立来源。
- 云端拉回的 token 事件会按原始来源合并到 Codex、OpenClaw、Claude Code 等来源里。
- 如果某个来源本机没安装 watcher，但远端有同步记录，首页会正常显示来源数据，并标注“来自其他设备数据”。
- 如果本机有历史记录但当前 watcher 没检测到安装，不再直接显示红色“未安装”，避免把历史或远端数据误导成不可用。

### 模型占比

- 事件级别的模型信息仍然默认只保存在本地，不随 token 事件上传。
- 同养小树同步会默认上传聚合模型占比，只上传按天、设备、来源、模型汇总后的 token 数。
- 设置里不再提供“同步聚合模型占比”开关，避免把同养小树同步拆成太多用户需要理解的选项。
- 有聚合模型统计时，远端拉回数据里的未归类模型占比可以被更具体的模型占比替换。
- 兼容旧迭代账本里“已同步但缺少 deviceId、同时仍保留本机模型名”的镜像行，避免这类历史数据漏传聚合模型占比。

### 排行榜公开偏好

- “公开使用偏好”开启后，常用模型优先来自同养小树的聚合模型统计。
- 这样 Windows / Mac 共同养树后，排行榜里的模型偏好不会只看当前设备本地有模型字段的事件。
- 仍然只上传最终聚合偏好，不上传单条会话、提示词、回复或本地路径。

### 设备信息

- Worker 新增 `tree_devices` 表，用来保存设备贡献摘要。
- 客户端同步时会上报粗粒度设备信息，例如 Mac、Windows、Linux，以及设备 id 和最后同步时间。
- 不上传 hostname、用户名、本地路径等更具体的设备身份信息。

### 按钮语义

设置里的“同养一棵树”现在只负责共享树档案：

- 未连接时显示“连接并同步小树”。
- 已连接时显示“同步小树”。
- 这个动作会同步 token 事件、设备 id、成就状态，以及按天汇总的聚合模型占比。

设置里的“全球排行榜”现在只负责排行榜隐私和公开偏好：

- 设置里保留“公开使用偏好”和数据安全说明。
- 加入、退出和刷新排行榜都放在排行榜页面顶部。
- 排行榜页未加入时显示“刷新榜单”和“加入排行”。
- 排行榜页已加入时显示“同步并刷新”和“退出排行榜”。
- 退出排行榜只删除远端排行榜数据，不删除同养小树云档。

### 单设备无感

- 没登录 GitHub 或没启用同养小树时，设置里不会显示聚合模型占比开关、设备贡献列表和高级云同步说明。
- 首次启动选择“新养一棵树”后，可以像普通本地应用一样使用，不需要理解云同步细节。

## Worker 和数据库

新增或更新的 Worker 能力：

- `/api/tree` 返回云端树事件、成就、设备摘要和聚合模型统计。
- `/api/tree/events` 支持空事件同步，用来只刷新设备快照。
- `/api/tree/events` 支持按设备替换聚合模型统计。
- 只要远端已有设备快照，即使还没有 token 事件或成就，也会被视为已有云端小树，方便新设备接入一棵刚开启同步的空树。
- 删除用户远端数据时，会同时清理新增的设备和模型统计表。


## 仍需要 Windows 或线上配合确认

- Windows 需要更新到包含本次修改的分支后重新同步。
- 线上 Worker 需要部署新表和新接口，否则设备列表和聚合模型占比不会从真实云端返回。
- 已经上传到旧 Worker 的 token 事件不会自动恢复事件级模型信息，因为旧同步本来没有上传模型字段。
- 如果 Windows 本地还保留详细历史记录，更新到新版客户端并同步后，可以把跨设备模型占比补成聚合视角。

## 当前产品判断

同养小树和排行榜应该继续保持两套语义：

- 同养小树是私有成长档案同步。
- 排行榜是公开或半公开的每日统计展示。

这样按钮虽然仍然分成两组，但每组只做一件事，用户不需要理解内部实现，也不会把“刷新排行榜”误解成“同步整棵树”。
