# Vibe Tree v0.7.1 发布文案草案

目标版本：`v0.7.1`。

这份文档分成两部分：

- App 内更新公告：来自 `updates/manifest.json`，给普通用户快速理解本次更新。
- GitHub Release Notes：正常 release 记录，给想了解完整版本变化的用户和开发者看。

## App 内更新公告

App 内公告不直接展示完整 GitHub Release body，而是读取 `updates/manifest.json` 里对应版本的结构化内容。

当前 `v0.7.1` 的 App 内展示内容大致如下，发布前需要用户确认：

```text
Vibe Tree v0.7.1

本次更新是 v0.7 社交和 macOS 菜单栏体验的补丁版：群组加入流程更完整，资料卡隐私可配置，群组排行榜口径更清楚，菜单栏浮窗的实时速率和排行榜展示也更稳定。升级后，本机小树和已有同步数据都会保留；社交、排行榜和同养小树仍然是可选功能。

主要变化
- 群组加入流程改成申请 / 审批机制：邀请码不再直接把人拉进群，群主或管理员可以统一处理待加入成员。
- 群主和管理员现在可以直接邀请好友入群，对方会在社交页看到邀请并选择加入或忽略。
- 资料卡新增隐私设置，可以控制谁能打开你的资料卡，以及是否展示等级、累计 Token、活跃天数和成就陈列。
- 群组排行榜支持在“总用量”和“入群后贡献”之间切换，同时继续保留 24h、7 天、30 天和全部时间范围。
- macOS 菜单栏浮窗继续打磨：实时速率图使用统一口径，排行榜标题明确为 24h，翻页和组件展示也更稳。
- 排行榜 7 天和 30 天范围现在会展示对应时间范围内有记录的用户，不再只显示当天活跃的人。

体验改进
- 修复全球榜和群组榜的 7 天 / 30 天人数异常偏少的问题。
- 修复当前版本内容在网络不稳定时可能显示英文 GitHub Release 正文的问题。
- 修复菜单栏实时速率图中端点圆形被拉伸、口径说明容易混淆的问题。
- 优化同养小树的模型聚合增量同步，减少不必要的重复快照。
- 补充社交和同步验证，覆盖群组申请、好友邀请、资料卡隐私、排行榜退出和隐私 payload 边界。

隐私说明
- 社交、排行榜和同养小树仍然都是可选功能。
- 资料卡隐私只控制聚合展示项，例如等级、累计 Token、活跃天数和成就。
- 群组榜和资料卡只使用 Token 数字、关系状态和聚合统计；不会上传提示词、回复内容、代码文件、本地路径或会话正文。

升级说明
- 升级不会清空本机数据。
- 排行榜本地缓存最长约 1 小时；如果刚升级后还看到旧榜单，稍后会自动更新，也可以使用排行榜页的刷新入口。
- 如果 macOS 菜单栏图标暂时看不到，可能是系统菜单栏空间不足，可以先收起不常用的状态栏图标。
```

## GitHub Release Notes

下面是建议放进 GitHub Release body 的正式版本记录。

```md
# Vibe Tree v0.7.1

## Added

- Added a group join-request flow so invite codes create pending requests for group leaders or officers to approve.
- Added direct friend-to-group invites that invited friends can accept from their Social page.
- Added profile-card privacy controls for visibility, level, token totals, active days, and achievement showcase fields.
- Added a group leaderboard basis switch between total usage and since-joining contribution.

## Changed

- Refined the macOS menu bar popover with clearer live-rate wording, a `24h leaderboard` label, improved paging behavior, and more polished component presentation.
- Improved group management wording around invites, requests, group usage sharing, and leaderboard contribution basis.
- Improved shared-tree model aggregate delta sync so model snapshots do not need to be resent in full on every delta response.

## Fixed

- Fixed global 7d and 30d leaderboards incorrectly requiring users to also have current-day activity.
- Fixed group 7d and 30d leaderboards using the same overly narrow current-day filter.
- Fixed the current-version dialog falling back to the English GitHub Release body when the remote manifest fetch is unavailable but a bundled manifest exists.

## Privacy

- Social features, shared-tree sync, leaderboards, public usage preferences, and profile-card fields remain optional.
- Profile-card privacy only exposes aggregate fields selected by the user.
- Vibe Tree does not upload prompts, replies, code file contents, local paths, session text, hostnames, or local usernames.

## Compatibility

- Existing local trees continue to work after upgrading.
- Existing `today` leaderboard requests remain supported as a compatibility alias for `24h`.
- The leaderboard cache may keep old standings locally for up to about one hour after upgrade; a fresh fetch will use the fixed range semantics.
```

## 发布时需要确认

- `package.json` 版本号和 tag 是否已经更新到 `0.7.1` / `v0.7.1`。
- `updates/manifest.json` 里的 `version` 和 `fullReleaseUrl` 是否对应真实 release tag。
- GitHub Release body 是否使用上面的正常 Release Notes，而不是 App 内公告。
- App 内「查看当前版本内容」是否能显示 manifest 中的短公告。
- 发布前需要用户确认 App 内短公告。
